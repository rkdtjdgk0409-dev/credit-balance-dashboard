# Hang on 신용잔고 → GitHub Pages → Notion 대시보드 v3

이 버전은 단순히 첫 번째 시계열 응답을 선택하지 않습니다.

1. Hang on 화면에 표시된 최신 기준일·신용비율·신용융자 잔고·고객예탁금을 먼저 읽습니다.
2. 네트워크 응답의 여러 시계열 후보를 모두 비교합니다.
3. 화면의 최신값과 일치하는 후보만 `data.json`으로 저장합니다.
4. 값이 맞지 않으면 작업을 실패시켜 잘못된 데이터가 Pages에 게시되지 않게 합니다.

## 교체할 파일

저장소 최상단에서 아래 파일을 새 버전으로 교체합니다.

- `index.html`
- `style.css`
- `app.js`
- `data.json`
- `update_data.py`
- `requirements.txt`
- `.nojekyll`
- `.github/workflows/main.yml`

저장소 최상단의 `update.yml`은 GitHub Actions가 읽지 않으므로 삭제해도 됩니다.

## 적용

1. 파일을 커밋합니다.
2. `Actions` → `신용잔고 데이터 자동 갱신` → `Run workflow`를 실행합니다.
3. 작업이 성공하면 `data.json`에 검증된 전체 시계열이 자동 커밋됩니다.
4. Pages는 보통 1~3분 뒤 갱신됩니다. 캐시가 남으면 주소 끝에 `?v=3`를 붙여 확인합니다.

## 화면 변경점

- 신용비율은 `신용융자 잔고 ÷ 고객예탁금 × 100`으로 브라우저에서 다시 계산합니다.
- 0 또는 비정상 금액은 표시하지 않습니다.
- 비교 차트는 신용융자와 고객예탁금의 축을 나눠 두 추세가 모두 보이게 했습니다.
- 지표 읽는 법을 요청한 4개 구간 문구로 교체했습니다.

## v5 수집 안정화

Hang on이 전체 차트 데이터를 한 번에 노출하지 않아도 작업이 실패하지 않습니다.
전체 시계열을 찾으면 그대로 사용하고, 찾지 못하면 페이지에 실제 표시된 기준일·신용융자 잔고·고객예탁금을 기존 `data.json`에 날짜별로 누적합니다.
